# RAMI (Right-Arm Mobile Intelligence) 프로젝트 개발 기획서

## 1. 프로젝트 개요
* **프로젝트명:** RAMI (라미 - 우익 암 모바일 지능형 로봇)
* **핵심 가치:** 스마트홈 허브나 스위치봇 등 별도의 가옥 내 인프라 구축 비용 없이, 기존 환경을 그대로 활용(Retrofitting)하여 문 개폐 및 전등 스위치를 유연하게 조작하는 모바일 매니퓰레이터 구현.
* **주요 개발 환경:** Window 10 Pro, MuJoCo 시뮬레이터, Python 3.10, Antigravity IDE (VS Code UI)
* **개발 시스템 사양:** CPU Intel i5-10400, RAM 32GB, GPU NVIDIA GeForce RTX 3080

---

## 2. 로봇(RAMI) 및 하드웨어 명세

### 2.1 기구부
* **모바일 베이스 (Base):** 
  * 메카넘 휠 기반의 전방향 이동이 가능한 납작한 동체. 
  * _(참고: 시뮬레이션 효율을 위해 외형은 원기둥 바퀴로 모델링하되, 키네마틱스 수식 또는 가상 관절로 전방향 이동 구현)_
* **리프트 기둥 및 로봇 암 (Arm):** 
  * 동체 중앙에 우뚝 선 기다란 기둥 구조.
  * 기둥 레일을 따라 상하 이동(Prismatic/Slide Joint) 및 수직축 중심 좌우 회전(Revolute/Hinge Joint)이 가능한 **단일 우측 암(Single Right-Arm)** 구조 채택.
  * 인간 주거 환경의 오른손잡이 표준 친화적 설계를 활용한 하드웨어 다이어트 및 제어 최적화.
* **말단 장치 (End-Effector):** 
  * 스위치 압봉 및 문고리 조작이 가능한 기본 그리퍼(Gripper).

### 2.2 센서 시스템
* **메인 센서:** 기둥 꼭대기 탑재 댑스 카메라 (Depth Camera) 
  * 상단 오버행 장애물(테이블, 침대 등) 회피 및 방 전체의 3D 포인트 클라우드(Point Cloud) 맵 생성.
* **서브 센서 1:** 동체 하단 8채널 초음파 센서 (모서리 4곳, 꼭지점 4곳) 
  * 발밑 사각지대 감지 및 비상 정지(Fail-safe)용 최단 거리 필터링 알고리즘 적용.
* **서브 센서 2:** 손목 카메라 (Eye-in-Hand) 
  * 스위치 QR 코드 인식 및 미세 조작 정밀 피드백.

## 3. 운영 시나리오 (Golden Path)
1. **사용자 가상 정의 데이터셋 구성:** 실제 앱/DB 개발을 우회하여, 스위치 정면 이미지 기준의 QR ID, 위치(침실1, 거실 등), 타겟 전등, 조작 방식(토글/버튼) 정보를 담은 가상 데이터(JSON) 사전 구축.
2. **Coarse 3D 매핑 (초기 스캔):** 모든 문이 열려있고 전등이 켜진 상태를 가정. RAMI가 집안을 주행하며 3차원 지도를 그리고, 스위치 형태의 미확인 오브젝트(`unverified`) 좌표를 맵에 마킹.
3. **Fine Identification (순회 및 동기화):** 매핑 완료 후 마킹된 스위치 좌표들을 순회. 손목 카메라로 QR 코드를 정밀 인식하여 사용자 데이터셋과 동기화.
4. **상관관계 학습:** 스위치를 직접 조작해 보며 천장 광원의 조도 변화(diffuse 값)를 감지, 스위치-전등 매핑 관계 학습.
5. **문 조작법 학습:** 방의 열려있는 문을 직접 여닫아 보며(문고리 하향 -> 동체 전진/후진 및 팔 밀기) 적절한 제어 프로필 학습.
6. **미션 수행:** 학습 완료 후 사용자가 특정 전등 제어 명령을 내리면 최적 경로 주행, 문 개폐, 스위치 정밀 조작을 순차적으로 자동 수행.

## 4. 선행 모듈별 개발 로드맵 (Unit Testing 순서)

### 📌 [Phase 1] 동체 움직임 구현 (우선 순위: 최상)
* **목표:** 원기둥 바퀴 4개가 달린 납작한 동체의 전방향 이동 메커니즘 검증.
* **검토 포인트:** 시뮬레이션 연산 효율을 위해 롤러 모델링을 생략하고, 메카넘 휠 키네마틱스 수식을 바퀴 모터에 매핑하거나 동체 자체에 가상 이동 관절(x, y, wz)을 달아 속도 제어 인터페이스 구축.

### 📌 [Phase 2] 로봇 암 및 엔드 이펙터 제어 구현
* **목표:** 리프트 기둥 상하 프리즘 관절, 팔의 회전 힌지 관절, 그리퍼 슬라이드 관절 배치 및 위치(Position) 제어기 검증.
* **검토 포인트:** 파이썬 타겟 좌표 입력 시 링커 블록과 팔이 지정된 위치로 부드럽게 도달하는지 확인.

### 📌 [Phase 3] 센서 시스템 구현
* **목표:** 기둥 상단 댑스 카메라, 손목 비전, 하단 초음파 센서(rangefinder)의 레이아웃 설정 및 데이터 추출.
* **검토 포인트:** GPU 기반 렌더링 부하 점검, 8개 초음파 레이저 빔 중 `min()` 필터링을 통한 최단 거리 근사값 추출 로직 검토.

### 📌 [Phase 4] 국산 스위치 조작법 학습
* **목표:** 한국식 시소형 똑딱이 스위치 모델링(커버+버튼 분리 및 힌지 제한)을 대상으로 정지 상태에서의 정밀 조작 궤적 획득.
* **검토 포인트:** 손가락 끝단 마찰력(Friction)과 스위치 스프링 탄성(Stiffness) 물리 속성 튜닝.

### 📌 [Phase 5] 문 및 문고리 조작법 학습
* **목표:** 문틀-문짝-문고리로 이어지는 계층 구조(Hierarchy Joint) 문 모델링을 대상으로 정지 상태에서 문 열기 연속 동작 구현.
* **검토 포인트:** 문고리를 내린 상태 유지 메커니즘 및 상태 머신(FSM) 순차 제어 코드 검증.

## 5. 예외 사항 및 개발 스코프 제한 (참고사항)
> [!NOTE]
> 본 프로젝트의 스코프를 명확히 하기 위해 아래와 같은 예외 사항을 둡니다.

* **QR 코드 부재:** QR 코드가 없는 스위치 대응 제외 (모든 타겟 스위치에 QR 부착 가정)
* **초기 매핑 방해:** 스캔 과정 중 닫힌 문 조우 대응 제외 (초기 매핑 시 모든 문 개방 가정)
* **비표준 광원:** 천장 외 공간(스탠드 등)의 광원 예외 제외 (모든 광원은 천장 위치 가정)
* **인식 범위 외:** 인식 범위를 벗어난 광원 매핑 예외 제외
